
# SocialSync Content Generation

**Package:** `@blkoutuk/social-sync`  
**Integration:** Admin Dashboard Module  
**Status:** Active (Beta)

## Overview

SocialSync is the dedicated AI content generation studio integrated into the `blkoutuk/comms-blkout` admin dashboard. It empowers content teams to generate high-fidelity images and video assets tailored for social media campaigns directly from the admin interface.

The module acts as a "Headless Studio," consuming tasks from automated agents (News Crawlers, Viral Trends Bots) and pushing generated assets back to the social automation pipeline.

## Key Features

-   **Multi-Model Support:** Leverages Google Gemini (Nano Banana Pro) for imagery and Veo for video generation.
-   **Agent Integration:** Receives structured `AgentTask` objects from the backend services (News, Events, Trends).
-   **Brand Compliance:** Enforces aspect ratios, logo overlays, and specific visual styles (e.g., Cyberpunk, Corporate, Minimalist).
-   **Automation Pipeline:** Direct "Push to Automation" workflow to queue assets for publishing via the `comms-blkout` backend.

## Integration Guide

To include this module in the main Admin Dashboard:

1.  **Repository Location:** Ensure this package is located at `/packages/social-sync` within the monorepo.
2.  **Route Configuration:**
    Add the following route to the main Admin router:
    ```javascript
    import { SocialSyncStudio } from '@blkoutuk/social-sync';
    
    // ... inside router config
    {
      path: '/admin/content-generation',
      element: <SocialSyncStudio />,
      meta: { requiresAuth: true, role: 'content_manager' }
    }
    ```
3.  **Environment Variables:**
    The module requires the following env vars to be passed or configured in the build context:
    -   `API_KEY`: Google GenAI API Key (Paid tier required for Veo/Pro models).
    -   `BLKOUT_API_ENDPOINT`: URL for the main comms backend (simulated in mock mode).

## Workflow

1.  **Inbox:** The "Inbox" tab displays pending content requests generated by background agents.
2.  **Creation:** Users select a task, which pre-fills prompts and settings. They can tweak the `prompt`, `style`, and `overlays`.
3.  **Generation:** Assets are generated client-side using the Google GenAI SDK.
4.  **Publishing:** The "Push to Automation" action sends the `GeneratedAsset` payload to the `https://api.blkoutuk.com/comms/v1/queue` endpoint (mocked in `services/integration.ts`).
